## Executive State

The important architectural cut is now done:

- `true-drive` is no longer lockstep.
- `true-drive` uses microcoded C64 CPU plus event/catch-up.
- `debug-lockstep` is an explicit diagnostic mode.
- `$DD00` catch-up happens at the `KernelBus` boundary.
- `IecBus.beforeC64Read` is gone from runtime code.
- Direct `.executeToClock(...)` use outside kernel strategy / drive internals is audited.

Do not reintroduce lockstep as the `true-drive` production path.

## Runtime Code Shape

### Modes

`src/runtime/headless/session-modes.ts`

- `true-drive`: traps off, microcoded on, lockstep off.
- `debug-vice-compare`: traps off, microcoded on, lockstep off, trace channels on.
- `debug-lockstep`: traps off, microcoded on, lockstep on.

Use `debug-lockstep` only for ablation/debugging.

### IntegratedSession

`src/runtime/headless/integrated-session.ts`

- `useMicrocodedCpu` no longer implies `useCycleLockstep`.
- `Cpu65xxVice` is installed independently from the lockstep scheduler.
- Non-lockstep microcoded stepping runs `executeCycle()` until instruction boundary.
- `true-drive` / `debug-vice-compare` set kernel mode to `true-drive`.

### Kernel Sync

`src/runtime/headless/kernel/event-catchup-strategy.ts`

- Owns true-drive `runCycles` and `runInstructions`.
- Advances C64 by stepping instructions.
- Drive advances via `catchUpDrive` at cross-domain points.

`src/runtime/headless/kernel/lockstep-strategy.ts`

- Diagnostic only.

`src/runtime/headless/kernel/headless-machine-kernel.ts`

- Routes `runCycles`, `runInstructions`, and `catchUpDrive` through `SyncStrategy`.
- Uses `EventCatchupStrategy` by default.
- Uses `LockstepStrategy` only if session has explicit lockstep scheduler.

### IEC / KernelBus

`src/runtime/headless/kernel/headless-kernel-bus.ts`

- C64 `$DD00` reads/writes call `kernel.catchUpDrive(8, ctx.clock)` before observing or mutating IEC state.
- Constructor-time CIA2 initial write is protected by a drive-ready guard.

`src/runtime/headless/iec/iec-bus.ts`

- No `beforeC64Read`.
- No hidden drive catch-up inside IEC.
- IEC remains cached VICE-style bus state.

## Important Warnings

1. Do not treat green KERNAL LOAD as proof that custom fastloaders work.
   The next failure is likely after KERNAL handoff.

2. Do not re-add `IecBus.beforeC64Read`.
   `$DD00` is a cross-domain access and belongs in `KernelBus`.

3. Do not call `drive.executeToClock` from session, bus, scripts, tools,
   or chip wrappers. Use `kernel.catchUpDrive`.

4. Do not make game-specific loader hacks.
   Any progress must identify the VICE/headless event now matched.

5. Lockstep is allowed only as explicit `debug-lockstep`.
   It is not acceptance for `true-drive`.
