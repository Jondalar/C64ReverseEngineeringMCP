# Spec 017: Clean Fixture Project For UI Testing

## Problem

BWC Reverse is the only large real project available, but it is too
chaotic to validate UI changes against. Every UI iteration drowns in
noise from years of accumulated state. The previous fixture
(Polarbear) was a commercial product and has been purged.

## Goal

A small, in-tree fixture project the UI can boot against — purely
synthetic, no commercial inputs, deterministic, and easy to rebuild
from a script. Suitable for browser smoke and for the workspace UI
runner panels (audit, repair, workflow, questions tab).

## Layout

```
fixtures/
  ui-smoke-project/
    .gitignore                # ignore knowledge/, views/, analysis/
    input/prg/sample.prg      # synthetic 6502 PRG (≤4 KB) we own
    knowledge/                # generated by the bootstrap script
    views/
    analysis/
    session/
```

The PRG must be ours: a tiny hand-written program that prints "HELLO"
and busy-loops, assembled with the bundled KickAssembler path so the
provenance is unambiguous.

## Bootstrap Script

Add `scripts/bootstrap-ui-fixture.mjs`:

1. Resolve `fixtures/ui-smoke-project`.
2. Wipe `knowledge/`, `views/`, `analysis/`, `session/`.
3. Initialize the project (`project_init` equivalent).
4. Run the PRG workflow (analyze + disasm + ram-report + pointer-report
   + import + build views).
5. Seed a handful of representative open questions / tasks /
   checkpoints so the dashboard panels and the upcoming Questions tab
   have content to show.

## Bytes Source

Generate the synthetic PRG once, commit it to the repo. Document the
KickAssembler source in the fixture so anyone can rebuild it:

```
fixtures/ui-smoke-project/src/sample.asm
```

`scripts/bootstrap-ui-fixture.mjs` does not re-assemble; it only runs
the workflow.

## UI Use

`npm run ui:serve -- --project fixtures/ui-smoke-project` boots the UI
with the fixture as the default project.

`package.json` gains:

```
"ui:fixture": "npm run ui:build && npm run ui:serve -- --project fixtures/ui-smoke-project"
```

## Acceptance Criteria

- A fresh checkout can run `node scripts/bootstrap-ui-fixture.mjs`
  followed by `npm run ui:fixture` and reach a populated dashboard
  with audit severity ok, ≥10 entities, ≥5 open questions, fresh
  views.
- The fixture is byte-for-byte ours and contains no commercial
  content.

## Tests

- A small smoke that runs the bootstrap script and asserts the
  resulting knowledge counts.

## Out Of Scope

- Multiple fixture projects (one is enough for now).
- A fixture for the runtime trace tooling.
