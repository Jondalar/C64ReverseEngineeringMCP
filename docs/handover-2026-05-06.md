## Pitfalls Hit / Lessons

1. **`git mv` after staged-but-not-committed `Edit`s loses banner edits.**
   The mv stages the rename with INDEX content, leaves working-tree mods
   unstaged. Always `git status` before commit; expect "Modified" lines after
   `git mv` if the file was edited beforehand.

2. **CIA2 microcoded CPU replacement (`true-drive` mode) requires kernel sync.**
   When `useMicrocodedCpu` swaps `c64Cpu`, kernel's `c64Cpu` field must
   also update — otherwise CIA `clkPtr` closure reads stale Cpu6510 cycles
   (= 0 forever) and AlarmContextCycled dispatch storm trips at clk=5000.

3. **KernelBus construction must happen BEFORE chip installs.**
   CIA2 install fires an initial `iecWrite(0xff, 0x3f)`; if `kernel.bus`
   isn't built yet, `Cannot read properties of undefined`. Solution:
   construct bus right after `iecBus + c64Bus`, before chip installs.

4. **Wrapping `kernel.bus.c64Write` in tests doesn't always intercept.**
   The "live routing" smoke for `$DD00` writes failed despite routing
   working in production. Cause inconclusive (probably reset semantics or
   instance vs prototype property). Skipped in favor of static-wiring check;
   live routing implicitly proven by MM 38KB byte-perfect.

5. **NodeNext ESM has no `require`.** Use proper imports for cross-class
   reference; circular-type imports work fine via `import type`.

6. **CET/CEST timezone now in global `~/.claude/CLAUDE.md`.**
   User added.
