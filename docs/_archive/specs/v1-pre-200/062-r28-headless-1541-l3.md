# Spec 062 — R28 L3: Headless 1541 drive emulation (full)

## Problem

R28 details: headless emulator currently traps KERNAL LOAD/SAVE only. Custom drive loaders (LISTEN/SECOND/CIOUT M-W → M-E → bit-bang fastloader) fall through to bare KERNAL serial code that spins on $DD00 IEC handshake forever. Stack underflow, drift to BRK chain, trace ends. Affects virtually every C64 commercial title 1985+.

Spec 062 implements R28 **directly at Level 3** (full drive 6502 + VIA + GCR), skipping L1 (IEC trap) and L2 (CPU + VIA stub) per user decision. Reasoning: existing GCR codec (`src/disk/gcr.ts`, 475 LOC, full encode/decode/sync) makes the L2→L3 incremental cost small, and L3 fully replaces VICE for runtime traces of loader / drive / DOS-level behavior. VICE remains the route for VIC/SID/gameplay-loop traces (Spec 063 scope).

### Step-loop wiring

```ts
// session-manager.ts (extended)
const driveCyclesPerC64Cycle = isPal ? 1000000 / 985248 : 1000000 / 1022727;
let driveAccum = 0;

function step() {
  const c64Cycles = c64Cpu.step();          // execute one C64 instruction
  iecBus.tickC64Side(c64Cpu.bus.read($DD00)); // propagate C64 → bus
  driveAccum += c64Cycles * driveCyclesPerC64Cycle;
  while (driveAccum >= 1) {
    const driveCycles = driveCpu.step();
    via1.tick(driveCycles); via2.tick(driveCycles); // timer countdown, IRQ check
    iecBus.tickDriveSide(via1.outputBits()); // propagate drive → bus
    driveAccum -= driveCycles;
  }
  if (recordTrace) traceStream.append({ cpu: "c64", pc: c64Cpu.pc, ... });
}
```

### IEC bit-mirror

```ts
// iec-bus.ts (open-collector wired-AND)
class IECBus {
  c64AtnOut = true;  c64ClkOut = true;  c64DataOut = true;   // pulled-high default
  driveAtnAck = true; driveClkOut = true; driveDataOut = true;

  // Line state = AND of all drivers (low wins).
  get atn(): boolean { return this.c64AtnOut; }            // C64 drives ATN one-way
  get clk(): boolean { return this.c64ClkOut && this.driveClkOut; }
  get data(): boolean { return this.c64DataOut && this.driveDataOut && this.driveAtnAck; }

  // C64 reads current bus state via $DD00 input bits.
  // Drive reads via VIA1 PB input bits (with pin-mapping).
}
```

### Drive ROM bundling

Repo path: `resources/roms/dos1541-325302-01+901229-05.bin`.

Loader:
```ts
const envPath = process.env.C64RE_1541_ROM_PATH;
const bundledPath = resolve(__dirname, "../../../../resources/roms/dos1541-325302-01+901229-05.bin");
const romPath = envPath ?? bundledPath;
if (!existsSync(romPath)) throw new Error(`1541 ROM not found at ${romPath}`);
const dos1541 = readFileSync(romPath);
if (dos1541.length !== 16384) throw new Error(`1541 ROM must be 16384 bytes; got ${dos1541.length}`);
```

ROM mapped at $C000-$FFFF in drive address space. Read-only; writes ignored.

### Sprint 62 — GCR drive-side I/O + write back + persist

Goal: drive reads sectors via $1C01 read-latch (head positioning via VIA2 control bits → G64 byte-stream lookup). Writes go to in-memory track buffer; persisted to `<image>_session.g64` on session-end or explicit save call.

Scope:
- `src/runtime/headless/drive/head-position.ts` — track number ↔ G64 track-byte-stream cursor
- `src/runtime/headless/gcr-drive/read-latch.ts` — $1C01 read returns next byte from current track at head position; SYNC-mark detection sets V flag (per real 6522 SO pin behavior tied to GCR sync)
- `src/runtime/headless/gcr-drive/write-buffer.ts` — drive write to $1C01 stages bytes in current-track buffer
- `src/runtime/headless/gcr-drive/session-persist.ts` — on session.stop or explicit `headless_drive_persist_writes`, writes modified buffer back as `<image>_session.g64` (G64 format preserved, only modified tracks updated)
- `src/runtime/headless/drive/via2-gcr.ts` extension: motor on/off (ignored for sim), head step (PB bits 0-1 step, advances track), write-protect sense
- Reuse existing `src/disk/gcr.ts` for byte-stream encoding/decoding helpers
- Synthetic test: drive code does standard SCAN routine (read track 18 sector 0 = BAM), assert correct BAM bytes returned. Then drive writes a modified BAM, persist, re-load, assert persisted file has modification.
- Smoke `scripts/sprint62-smoke.mjs`

Tests: head position update, $1C01 read returns track bytes, write buffer round-trip, session-persist creates `<image>_session.g64` with modifications.

### Sprint 64 — VSF (VICE Snapshot Format) read + write

Goal: bridge VICE ↔ headless via shared snapshot format. Save full headless state as VSF; load VSF (from VICE or own previous run) and resume from that point.

**Why this matters:**
- Cross-emulator workflow: capture interesting state in VICE (where human can play interactively), snapshot, hand to headless for cycle-exact RE / trace / scripted experimentation.
- Headless → headless: long-running boot sequence saved once, loaded for fast-iteration on subsequent investigations (skip 30-second loader replay).
- Spec 063 forward compat: as VIC/SID/CIA modeling lands, VSF coverage grows automatically.

**VSF format reference:**
- Format: chunked (4-byte magic "VICE" + version + module chunks). Each module is a name + version + length + binary blob.
- Module names match VICE source: "MAINCPU", "C64MEM", "VIA1d1541", "VIA2d1541", "DRIVECPU", "GCR", etc.
- VICE source `src/snapshot.c` + per-module `*.c` snapshot read/write functions are the canonical reference. Project remains MIT — snapshot module structure is documented public format, no code lifted.

**Scope:**

Modules headless WRITES (post-Sprint 60-63):
- C64 MAINCPU: A/X/Y/SP/PC/P, current cycle counter
- C64 RAM: $0000-$FFFF (64KB) + COLOR ($D800-$DBFF, 1KB)
- DRIVECPU: A/X/Y/SP/PC/P, drive cycle counter
- DRIVE RAM: $0000-$07FF (2KB)
- VIA1d1541: register state + timers + IFR/IER + shift register state
- VIA2d1541: register state + timers + IFR/IER + shift register state
- IEC: bus pin state (ATN/CLK/DATA + each driver's contribution)
- GCR head: current track + byte position within track + write buffer for modified tracks

Modules headless can READ from a VSF but **ignores** (until Spec 063 lands the corresponding peripheral, then auto-promoted to load):
- VIC, SID, CIA1, CIA2 (partial — IEC bits load), KEYBOARD, JOYSTICK, TAPE, EXPANSION

On VSF load:
- Parse all chunks, log each module: `loaded` / `ignored (not modeled — Spec 063)` / `error (unknown module)`.
- Apply loaded modules to current headless state; ignored modules leave that subsystem at its default boot state. Emit warning if VSF was saved by VICE with VIC/SID register state that headless can't restore.

On VSF save:
- Write only modules headless owns. Saved file is loadable by VICE (which fills missing modules with defaults — same pattern in reverse).

**Sprint 64 work:**
- `src/runtime/headless/vsf/vsf-format.ts` — chunk parser + writer (magic, version, module chunks, length-prefixed blobs)
- `src/runtime/headless/vsf/module-mapping.ts` — table from VSF module name → headless serializer/deserializer
- Per-module serializers: `serializeMainCpu`, `serializeC64Memory`, `serializeDriveCpu`, `serializeDriveRam`, `serializeVia1d1541`, `serializeVia2d1541`, `serializeIecBus`, `serializeGcrHead`. Each ~30-50 LOC.
- New MCP tools:
  - `headless_session_save_vsf(path)` — write current state
  - `headless_session_load_vsf(path)` — replace state from file (warns on ignored modules)
- Tests: round-trip (save → load → state matches), VSF written by Sprint 64 loadable in VICE, VSF written by VICE loadable in headless (with expected ignored-module warnings)
- Smoke `scripts/sprint64-smoke.mjs`

**Deliverable:** headless VSF I/O works for the modeled subset. Cross-emulator workflow possible.

**Forward compat to Spec 063:** every Spec 063 sprint that lands a new peripheral (VIC, SID, CIA1, etc.) adds the corresponding `serialize*` function to module-mapping. VSF coverage grows monotonically until full VICE parity.

## Out of scope (Spec 063 territory)
